This page loads a simple applet which tests the following Java code which tries to get access to the browsers window object 

  JSObject win = JSObject.getWindow(this);
  System.out.println("WINDOW IS: " + win)

If successful the Java console will display something like: 

 WINDOW IS: [object DOMWindow]

On FireFox 3.6.11-12 on Mac OS X JSObject.getWindow returns null. 

 WINDOW IS: null

Here's the Java code for the applet: 

  package org.concord.applet;

  import javax.swing.*;
  import netscape.javascript.JSObject;
  import netscape.javascript.*;

  public class TestJApplet extends JApplet {
      public void init() {
          JSObject win = JSObject.getWindow(this);
          System.out.println("WINDOW IS: " + win);
      }
  }

See: Mozilla Bug 606737: Java Applet: JSObject.getWindow(this) returns null

Looking at the changesets in the source as FiresFox progressed from 3.6.10 to 3.6.11 this one by Steven Michaud on Sep 24, 2010 stands out: Bug 598453 - r=josh a1.9.2.11=clegnitto. The commit log message is obscure but the commit bumps these two artifacts from the Java Plugin Project: Java Embedding Plugin and the MRJ Plugin 1.0-JEP from version 0.9.7.3 to 0.9.7.4. Unfortunately the latest commit to the sourceforge CVS repository is from Mar 09 2010 and is for the v0.9.7.3 release. In addition the content on the page with Mozilla Bug 598453 is not visible to unauthorized developers. This makes me wonder if the update was a security fix.
